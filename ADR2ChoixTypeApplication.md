## ADR Type d'application

### Titre:
Choix du type d'application

### Contexte:
L'application est destinée à un public non technique, cette application doit être simple, pratique et adaptée à leurs besoins spécifiques. 

### Décision:
On a choisis une application web et mobile. 
Vu que c'est un outil pour les collaborateurs d'une entreprise, on peut avoir l'application sous format web et mobile. 

L'utilisation de Flutter permet de maximiser le retour sur investissement (ROI) : pour le prix d'un développement quasi unique, nous couvrons 100% des usages collaborateurs (bureau et mobilité) (ADR choix de stack) que si on choisis une application web responsive avec React par exemple.

### Conséquences:

Maintenance simplifiée mais addaptation du code pour les différents écrans a gérer.
